# distill-lab

A small PyTorch framework for teacher-student language model distillation.

The repo focuses on a practical workflow: train or load a stronger causal LM teacher, distill it into a smaller custom student, then compare teacher vs scratch student vs distilled student on quality, latency, and memory.

## What this project does

- trains local teacher and student causal language models
- distills teacher logits into a smaller student with KL + cross-entropy + optional hidden-state matching
- supports local tokenizers and Hugging Face tokenizers
- supports local teacher checkpoints and frozen Hugging Face causal LM teachers such as `gpt2`
- summarizes runs into Markdown tables, CSV files, SVG learning curves, and benchmark reports

## Quick start

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Main use cases

### GPT-2 teacher -> custom student

Use a pretrained open-source teacher with the teacher tokenizer, then train a smaller custom student on Tiny Shakespeare:

```bash
python -m src.train --config configs/student_scratch_shakespeare_gpt2.yaml
python -m src.train --config configs/student_distill_shakespeare_gpt2.yaml
python scripts/benchmark_runs.py \
  results/student_scratch_shakespeare_gpt2 \
  results/student_distill_shakespeare_gpt2 \
  --labels scratch distill \
  --output-dir results/reports/shakespeare_gpt2_bench
```

This is the closest path to a real public-portfolio distillation example using an open model.

### Local teacher -> local student

Train a teacher locally, distill into a smaller student, and compare all three runs:

```bash
python -m src.train --config configs/teacher_shakespeare_bpe.yaml
python -m src.train --config configs/student_scratch_shakespeare_bpe.yaml
python -m src.train --config configs/student_distill_shakespeare_bpe.yaml
python scripts/summarize_runs.py \
  results/teacher_shakespeare_bpe \
  results/student_scratch_shakespeare_bpe \
  results/student_distill_shakespeare_bpe \
  --labels teacher scratch distill \
  --output-dir results/reports/shakespeare_bpe
python scripts/benchmark_runs.py \
  results/teacher_shakespeare_bpe \
  results/student_scratch_shakespeare_bpe \
  results/student_distill_shakespeare_bpe \
  --labels teacher scratch distill \
  --output-dir results/reports/shakespeare_bpe_bench
```

### Zero-friction smoke test

If you just want to verify the pipeline quickly:

```bash
python -m src.train --config configs/teacher_toy.yaml
python -m src.train --config configs/student_scratch_toy.yaml
python -m src.train --config configs/student_distill_toy.yaml
```

## Open datasets

Prepare larger public corpora with the included scripts:

```bash
python scripts/prepare_wikitext2.py
python scripts/prepare_tinystories.py --source-dir /path/to/TinyStories
```

Then run the matching configs in `configs/` for WikiText-2 or TinyStories.

## Reporting and benchmarking

Training runs write `metrics.jsonl`, checkpoints, and `run_config.yaml`. After that you can generate:

- quality summaries with `scripts/summarize_runs.py`
- latency and memory summaries with `scripts/benchmark_runs.py`

Those reports produce Markdown tables, CSVs, and SVG plots suitable for a README or portfolio writeup.

## Repo map

```text
configs/   experiment configs
scripts/   dataset prep, reporting, benchmarking
src/       training, evaluation, tokenization, teachers, losses
data/      local corpora
artifacts/ saved tokenizer models
results/   checkpoints and metrics
```

## Current scope

- Hugging Face teacher support is currently focused on causal language models.
- The GPT-2 path requires model/tokenizer files to be downloadable or already cached locally.
- The repo is strongest today as a distillation experimentation framework, not a production inference package.
